Ladybird adopts Rust, with help from AI

The Ladybird browser project, led by Andreas Kling, has announced a strategic shift to adopt Rust for key components, specifically targeting its LibJS JavaScript engine, after previously exploring Swift. This move stems from Rust's mature ecosystem and memory-safety guarantees, addressing long-standing concerns with C++'s vulnerability to memory errors in critical systems like browsers. The decision marks a pragmatic pivot from earlier evaluations that rejected Rust due to its poor fit for C++-style object-oriented programming, which is prevalent in web platform architectures with deep inheritance hierarchies and garbage collection. By choosing Rust, Ladybird aligns with industry trends seen in Firefox and Chromium, which have incrementally integrated Rust, highlighting a broader industry shift towards memory-safe languages for security-critical applications. The porting effort is not a full rewrite but a gradual migration, reflecting a nuanced approach to balancing innovation with stability in a complex, long-lived codebase.

Implementation details reveal a highly methodical process: the initial port of LibJS‚Äîcomprising lexer, parser, AST, and bytecode generator‚Äîwas accomplished using AI tools like Claude Code and Codex, but with heavy human direction through hundreds of small prompts to steer the translation. A core requirement was byte-for-byte identical output between the C++ and Rust pipelines, enabling lockstep testing where both run simultaneously to diff outputs and catch bugs immediately. This resulted in 25,000 lines of Rust code ported in approximately two weeks, a task estimated to take months manually, with zero regressions across extensive test suites including 52,898 test262 cases and 12,461 Ladybird regression tests. The Rust code intentionally mimics C++ patterns, such as register allocation, to ensure compatibility, though it is acknowledged as non-idiomatic Rust, with plans for future cleanup once the C++ pipeline is retired. Development will continue in C++ for new features, with Rust ports proceeding as a sidetrack, managed by the core team to avoid fragmentation, emphasizing a cautious, interoperability-focused strategy.

This story resonates deeply with the Hacker News audience due to its implications for software engineering practices and AI's evolving role. It showcases how AI-assisted code translation, when combined with rigorous testing and human oversight, can accelerate migrations that were once prohibitively costly, potentially lowering barriers to adopting safer languages in legacy systems. Societally, it contributes to the push for memory safety in critical software like browsers, which could enhance security against exploits, aligning with regulatory trends. Long-term, it signals a shift in developer workflows, where AI becomes a collaborative tool for refactoring rather than autonomous coding, raising questions about skill evolution and code quality. HN users care intensely about such technical decisions, debating language choices, project sustainability, and the ethics of AI in coding, making this a focal point for discussions on innovation versus pragmatism in open-source development.

These methods necessitate extensive data collection and retention, clashing with fundamental privacy principles like data minimization and purpose limitation embedded in laws such as GDPR. The tension is not merely operational but structural, as enforcement pressures prioritize defensible evidence over privacy-preserving designs, pushing technology towards increased surveillance and data hoarding.

Implementation details highlight the pervasive nature of this issue. Major platforms like Meta use facial age estimation on Instagram through third-party partners, requiring video selfies for AI analysis, while TikTok scans public videos for age inference. Google and YouTube rely on behavioral signals, escalating to ID or credit card checks when uncertainty arises. In regions like Brazil, where the Statute of Child-rearing and Adolescents enforces strict verification, weak identity infrastructure forces providers to depend on facial estimation and offshore vendors, exacerbating surveillance. Similarly, Nigeria sees increased biometric inference due to a lack of formal IDs. Data points reveal common failures: false positives locking adult accounts, false negatives allowing underage access, and the imperative to retain verification logs for regulatory defense, which amplifies breach risks. The article notes that 'to prove they are following age-verification rules, platforms must log verification attempts, retain evidence, and monitor users over time,' illustrating the inherent data retention demand.

Societally, age-verification systems risk reshaping internet identity and access, potentially normalizing pervasive monitoring for all users and eroding online anonymity.
